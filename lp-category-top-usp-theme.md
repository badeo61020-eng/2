# LP Category Theme

## Category Order

- `TOP`
- `USP`
- `問題点`
- `解決策1`
- `解決策2`
- `解決策3`
- `use`
- `仕様`

## Meaning of Each Category

### TOP

- First-view section
- Main hook
- Strongest impression
- Catch copy and sub copy

### USP

- Unique selling proposition
- Biggest difference from competing products
- Core reason this product should be chosen

### 問題点

- Customer pain points
- Dissatisfaction with common products
- Situations where existing options feel insufficient

### 解決策1

- First main product strength
- Usually the clearest or most marketable function

### 解決策2

- Second main product strength
- Often a supporting practical or emotional benefit

### 解決策3

- Third main product strength
- Can reinforce trust, durability, usability, or differentiation

### use

- Usage scenes
- Lifestyle fit
- Who uses it and when

### 仕様

- Specs
- Material
- Size
- Detail information
- Purchase decision support

## Practical Rule

- This format is suitable for EC LPs that need a clear visual block structure.
- It can be used instead of PASBECONA when the user wants a more section-based layout.
- The highest-priority blocks in this format are:
- `USP`
- `問題点`
- `解決策`
- These three should always be clarified first before expanding the rest of the LP.
- Repeated or very similar content across these three blocks must be removed.
- `USP` should explain only the core unique difference.
- `問題点` should explain only customer pain points and dissatisfaction.
- `解決策` should explain only the product's solutions and selling points.
- The writing should still follow the same core rules:
- match the real product
- use PMM thinking first
- reflect image-based judgment
- reflect EC research
- reflect review analysis
- reflect keyword thinking
